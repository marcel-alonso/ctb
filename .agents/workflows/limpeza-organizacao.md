# Workflow: Limpeza e Organização

**Status:** Ativo  
**Instruções para Manutenção do Repositório**

---

## 1. Arquivamento de Relatórios

Sempre que um relatório de análise, review ou sumário perder a relevância imediata, ele deve ser movido para:
- `docs/_archive/`

Isso evita o "ruído de contexto" na raiz do projeto.

## 2. Remoção de Lixo Técnico

- Arquivos `connection_test.txt`, `debug.txt`, `Visual_FINAL.txt` e similares devem ser removidos após o uso.
- Pastas temporárias de build não devem ser versionadas.

## 3. Padronização de Nomes

- Seguir padrão Kebab-case para arquivos e pastas.
- Imagens em `/assets/images/` com nomes descritivos (ex: `hero-bambu-luxo.webp`).

## 4. Frequência de Limpeza

Realizar uma varredura de limpeza a cada ciclo de grandes atualizações.
